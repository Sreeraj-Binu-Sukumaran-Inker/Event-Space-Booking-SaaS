# Repository Guidelines

## Project Structure & Module Organization
- `src/server.ts`: application entrypoint and middleware wiring.
- `src/routes`: route registration (`*.routes.ts` / `*.route.ts`).
- `src/controllers`: HTTP handlers and response shaping.
- `src/services`: business logic and Prisma-backed operations.
- `src/middleware`: auth, tenant, role, and global error middleware.
- `src/prisma/client.ts`: shared Prisma client instance.
- `prisma/schema.prisma`, `prisma/migrations/*`, `prisma/seed.ts`: database schema, migration history, and seed data.
- Build output goes to `dist/` (generated by TypeScript).

## Build, Test, and Development Commands
- `npm install`: install dependencies.
- `npm run dev`: run API with `ts-node-dev` (auto-reload).
- `npm run build`: compile TypeScript to `dist/`.
- `npm start`: run compiled server from `dist/server.js`.
- `npx prisma migrate dev`: create/apply local migrations.
- `npx prisma db seed`: run seed script (`prisma/seed.ts`).

## Coding Style & Naming Conventions
- Language: TypeScript (`strict` mode enabled via `tsconfig.json`).
- Indentation: 2 spaces; keep semicolons and double quotes consistent with existing files.
- Filenames: use lowercase domain names with suffixes: `*.controller.ts`, `*.service.ts`, `*.middleware.ts`, `*.routes.ts`.
- Exports: prefer named exports for classes/services; default export for route modules.
- Keep controllers thin; move business/database logic into services.

## Testing Guidelines
- No automated test framework is currently configured in `package.json`.
- Minimum validation before PR: `npm run build` and smoke-test key endpoints locally.
- When adding tests, place them near source as `*.test.ts` or under `src/__tests__/`, and prioritize service + auth flows.

## Commit & Pull Request Guidelines
- Git history is not available in this folder snapshot; use Conventional Commits going forward (`feat:`, `fix:`, `refactor:`, `chore:`).
- Keep commits focused and scoped (one concern per commit).
- PRs should include: summary, affected routes/modules, migration notes (if `prisma/schema.prisma` changes), and local verification steps.
- Include sample request/response payloads for API behavior changes.

## Security & Configuration Tips
- Required env values are loaded from `.env` (`DATABASE_URL`, JWT secrets, CORS origin, `PORT`).
- Never commit secrets or production credentials.
- Set cookie `secure: true` and production-safe CORS settings before deployment.
